# Prompt Contract Template

## Objective
- 達成すべき結果を 1 文で書く

## Inputs
- issue、task brief、spec、test、verify command など必要 input を列挙する

## Constraints
- 守るべき public contract、scope、更新必須 artifact を書く

## Forbidden Actions
- 推測でやってはいけない変更、混ぜてはいけない作業を書く

## Missing Information Policy
- 何が足りなければ停止するか
- 仮定を置いて進める場合に何を報告へ残すか

## Completion Criteria
- 完了判定に使う verify 可能な条件を書く

## Output Format
- 最終報告の見出しを固定する
