# 読書案内

この読書案内は、本文と repo artifact を読んだあとに、どの source を足すと理解が深まるかを整理したものである。網羅よりも、Prompt / Context / Harness の主要論点を支える durable な source を優先する。

## 使い方

- 最初に戻る先は常に本書の本文と repo artifact である
- その次に、使っている tool や platform の official docs を読む
- 書籍や handbook は、組織へ持ち帰るときの判断軸を補強するために使う
- ここで挙げる source は、学術的な完全性より再利用価値を優先している

## Prompt と要求定義

| source | 何を補強するか | 関連章 |
|---|---|---|
| 利用中モデルの official prompting guide | Prompt Contract、制約設計、tool use の前提 | CH02, CH04 |
| 利用中モデルの official eval docs | case 設計、rubric、version comparison | CH04 |
| Michael Nygard の Architecture Decision Records (ADR) | 決定理由を短く残す書き方 | CH03 |
| 組織内の product spec / acceptance criteria template | exploratory dialogue から実装準備ができた artifact へ落とす基準 | CH03 |

## 使い分けの原則

- Prompt の議論で迷ったら、まず official docs と local eval に戻る
- Context の議論で迷ったら、repo artifact と ownership の正本に戻る
- Harness の議論で迷ったら、verify、evidence、review policy の順に戻る
- 書籍や handbook は、local artifact の代わりではなく、判断軸を補うために使う
