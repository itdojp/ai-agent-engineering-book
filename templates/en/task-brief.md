# Task Brief Template

## Source
- Record the starting issue, seed issue, PR, or other originating artifact.

## Goal
- State the result this task must achieve in one sentence.

## Scope
- Name only the changes included in this run.

## Inputs
- List the specs, ADRs, code, tests, and docs that must be read.

## Deliverables
- List the artifacts that must be updated.

## Constraints
- Record the public contracts, non-functional limits, and required update order that must be preserved.

## Acceptance Criteria
- Write the conditions that let a reviewer verify the result.

## Verification
- List the verify commands that must be run.

## Open Questions
- Keep unresolved points here without creating a competing source of truth.

## Out of Scope
- State what this run must not do.

## Suggested Prompt
- If useful, record the smallest prompt that can hand this task to a coding agent.
