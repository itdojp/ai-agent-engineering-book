# manuscript/AGENTS.md

## Purpose
書籍原稿の標準構成と文体ルールを定義する。

## Style
- 日本語
- 実務書として簡潔に書く
- 主張 → 理由 → 例 → artifact の順で説明する
- 不必要な比喩を避ける
- 各章で bad / good example を最低 1 組入れる
- 演習は 2 問とする

## Chapter Contract
各章には次を含める。
- 導入: この章の位置づけ
- 学習目標: 2〜3 項目
- 小見出し: brief に従う
- 参照 artifact の説明
- 章末の `Source Notes / Further Reading`
- 演習: 2 問
- 章末まとめ

## Cross-reference
- 前提章がある場合は brief の dependencies を確認する
- glossary に定義済みの語を優先する
- sample-repo の artifact 参照は相対パスをそのまま書く
