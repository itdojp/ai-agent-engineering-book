# Task Brief Template

## Source
- 起点 issue、seed issue、PR、会話ログなどを書く

## Goal
- この task の達成結果を 1 文で書く

## Scope
- 今回含める変更だけを書く

## Inputs
- 読むべき spec、ADR、code、tests、docs を列挙する

## Deliverables
- 更新対象 artifact を列挙する

## Constraints
- public contract、非機能制約、更新順序などを書く

## Acceptance Criteria
- verify 可能な受け入れ条件を書く

## Verification
- 実行すべき verify command を列挙する

## Open Questions
- source of truth を増やさずに残す未解決事項を書く

## Out of Scope
- 今回やらないことを書く

## Suggested Prompt
- 必要なら、この task を coding agent に渡す最小 prompt を書く
