# Progress Note Template

## Status
- Optional. Record a coarse state such as `in-progress`, `blocked`, or `ready-for-handoff`.

## Current Goal
- State the current work package in one sentence.

## Completed
- List what finished in this session.

## Decided
- Record what was firmly decided in this session.

## Open Questions
- List what remains unresolved.

## Last Verify
- Record the latest verify command and its result.

## Changed Files
- List the diffs that a reviewer or handoff owner should track.

## Resume Steps
- Record which artifacts to read first and in what order when work resumes.

## Next Step
- State the next single work package.
